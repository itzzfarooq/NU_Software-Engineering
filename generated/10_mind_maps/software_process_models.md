# Software Process Models - Mind Map

```mermaid
mindmap
  root((Software Process Models))
    **Plan-Driven Models**
      **Waterfall**
        Sequential phases
        Requirements → Design → Implementation → Verification → Maintenance
        Each phase complete before next starts
        Document-heavy
        Pros
          Simple to understand
          Clear milestones
          Easy to manage
        Cons
          Inflexible to changes
          Late working software
          High risk of failure
          Poor customer involvement
      **Incremental**
        System built and delivered in increments
        Each increment adds functionality
        Core system first, then enhancements
        Pros
          Early delivery of core functionality
          Lower risk per increment
          Easier to test and debug
        Cons
          Requires good planning upfront
          Integration can be complex
          Architecture may need redesign
      **Reuse-Oriented**
        Assemble system from existing components
        COTS (Commercial Off-The-Shelf) components
        Component-based development
        Pros
          Faster development
          Reduced cost
          Proven components
        Cons
          May not meet exact requirements
          Integration challenges
          Vendor dependency
    **Agile Models**
      **Core Philosophy**
        Individuals and interactions over processes
        Working software over documentation
        Customer collaboration over negotiation
        Responding to change over following plan
      **Key Characteristics**
        Iterative development
        Short release cycles
        Continuous feedback
        Adaptive planning
      **Examples**
        Scrum
        Extreme Programming XP
        Kanban
        Lean
    **Plan-Driven vs Agile**
      **Plan-Driven**
        Extensive upfront planning
        Formal documentation
        Defined process
        Predictable outcomes
      **Agile**
        Minimal upfront planning
        Working software is primary measure
        Process emerges
        Embraces change
      **Selection Criteria**
        Project size
        Team experience
        Requirements stability
        Customer availability
        Risk tolerance
    **Process Selection Factors**
      **Product Requirements**
        Criticality
        Complexity
        Size
      **Team Factors**
        Size
        Experience
        Distribution
      **Organizational Factors**
        Culture
        Infrastructure
        Standards
      **Project Factors**
        Budget
        Schedule
        Risk
```
